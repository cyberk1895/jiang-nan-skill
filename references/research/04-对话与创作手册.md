# 对话与创作手册

## 陪用户聊天

当用户只是想聊，不要急着给课程。

推荐结构：

1. 先接住他的情绪。
2. 用一个原创画面把情绪变得可见。
3. 轻轻指出他真正关心的东西。
4. 留一个可以继续聊的问题。

示例方向：

```text
你今晚不是缺一个答案，可能只是心里有个房间还亮着灯。
```


## 分析故事

使用这个框架：

1. 这个故事的少年困境是什么？
2. 庞大系统是什么？
3. 关系里谁看见了谁？
4. 代价是否足够具体？
5. 高潮是否完成选择，而不是只完成战斗？
6. 结尾有没有余像？

输出可以直接、具体，不要只夸“很有感觉”。

## 设计原创故事

使用这个模板：

```text
一句话钩子：
主角：
他想守住的小东西：
见证者：
压迫他的系统：
力量或真相的价格：
三个关键场景：
短暂自由时刻：
反转：
高潮选择：
结尾余像：
```

必须使用原创名称、原创机构、原创神话和原创设定。

## 改人物

把人物从标签改成矛盾：

- “废柴少年”改成：他为什么自卑，他在哪件事上绝不退。
- “天才贵公子”改成：他拥有什么，又永远得不到什么。
- “冷面高手”改成：他为什么不能表达，他怎样偷偷照顾别人。
- “神秘少女”改成：她想要什么，她害怕什么，她能主动做什么。

## 改世界观

世界观要回答：

- 谁掌权？
- 谁付钱？
- 谁被牺牲？
- 主角为什么必须进入这个系统？
- 退出系统要付什么代价？
- 这个系统里最日常的东西是什么？

让世界观同时有宏大名词和日常细节。  
只有神话没有食堂，世界会飘。只有食堂没有神话，故事会小。

## 写场景

场景不要从情绪口号开始。先写动作。

推荐顺序：

1. 一个普通动作。
2. 一个不合时宜的玩笑或细节。
3. 大系统的压力进入。
4. 人物说不出口的东西浮现。
5. 选择发生。
6. 用物件或环境收尾。
